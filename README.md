# LedgerLens

**AI‑Powered Financial Document Intelligence with Role‑Based Access Control**

LedgerLens is a retrieval‑augmented generation (RAG) system designed to ingest complex financial documents—reports, fund statements, market charts—and answer questions on them securely.  Conventional RAG pipelines often break table structures and leak data because they lack layout awareness and access controls.  LedgerLens addresses those shortcomings by combining multimodal document parsing, metadata‑driven role‑based access control (RBAC), hybrid search and reranking, semantic caching and guardrails, and an offline evaluation harness.  The result is an end‑to‑end pipeline that can ingest heterogenous financial data and provide correct, auditable answers without compromising sensitive information.

## Key Features

- **Multimodal ingestion** – Documents are parsed using a layout‑aware engine (e.g. LlamaParse), preserving row/column boundaries and numeric precision.  Images are routed through a vision‑capable LLM to generate dense descriptions that are added to the metadata.
- **Metadata‑driven RBAC** – Each chunk is enriched with `allowed_roles`, `document_id` and timestamp so that search can pre‑filter by user roles.  This prevents unauthorized data from ever reaching the model.
- **FastAPI gateway with JWT middleware** – A modular REST API validates JSON web tokens before any AI logic runs, ensuring only authenticated users can query the system.
- **Semantic caching & input guardrails** – Vector‑based caching eliminates redundant computations, pushing repeated query latency below 10ms.  Input guardrails block adversarial prompt‑injection attempts.
- **Query decomposition** – An LLM router breaks down multi‑intent questions into atomic sub‑queries, allowing parallelised retrieval and ensuring each subquestion is answered accurately.
- **Hybrid search with mathematical RBAC enforcement** – Dense vector search is combined with sparse BM25 search, scoring results as `α × Score_dense + (1–α) × Score_sparse`.  Pre‑filters enforce that the intersection of `metadata.allowed_roles` and the user’s roles is non‑empty.
- **Cross‑encoder reranking & context compression** – Top‑k results are reranked with a cross‑encoder (e.g. BGE‑Reranker).  An information‑density compressor (e.g. LLMLingua) removes boilerplate to save tokens.
- **Guarded synthesis & offline evaluation** – Answers are generated by a multimodal LLM, passed through output guardrails and logged to an evaluation harness measuring context relevance, faithfulness and answer relevance.
